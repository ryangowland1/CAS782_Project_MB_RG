# VIATRA Stream Integration

This document explains how to connect VIATRA to the scene graph stream generated by:

- `src/scenegraph_stream_bridge.py`

## Stream Artifacts

The stream writer produces:

- `data/stream/latest_snapshot.xmi`: latest full scene graph
- `data/stream/snapshots/snapshot_XXXXXX.xmi`: history of snapshots
- `data/stream/events.jsonl`: append-only event stream with node/edge add-update-remove diffs
- `data/stream/current_state.json`: most recent event
- `data/stream/live_view.html`: auto-refresh browser visualization

## Recommended Eclipse Project Setup

1. Create an Eclipse Modeling project.
2. Import these resources:
   - `model/SceneGraph.ecore`
   - `queries/scenegraph.vql`
   - `data/stream/latest_snapshot.xmi`
3. Install VIATRA Query and Transformation SDK if not already installed.

## Incremental Query Matching Workflow

1. Start the stream bridge.
2. Keep `latest_snapshot.xmi` loaded in Eclipse.
3. Use VIATRA Query tooling to execute patterns from `queries/scenegraph.vql`.
4. As snapshots refresh, reload the model resource to update match sets.

For fully in-memory incremental updates in one model instance:

- Build a small JVM watcher that reads `events.jsonl` and mutates EMF objects (add/update/remove) directly.
- Attach VIATRA query engine to that EMF resource once, then consume match deltas incrementally.

## Event Schema (`events.jsonl`)

Each line is JSON with fields:

- `timestamp`
- `tick`
- `snapshot`
- `node_count`
- `edge_count`
- `node_changes`:
  - `added`: list of full node objects
  - `removed`: list of prior node objects
  - `updated`: list of `{before, after}`
- `edge_changes`:
  - `added`: list of `{type, source, target}`
  - `removed`: list of `{type, source, target}`

## Run Commands

Start mock stream:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run_scenegraph_stream.ps1 -Mock -Interval 1.0
```

Start live stream from CARLA:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run_scenegraph_stream.ps1 -Interval 1.0 -CarlaAddress 127.0.0.1 -Port 2000
```

Open visual output:

- Open `data/stream/live_view.html` in a browser.
